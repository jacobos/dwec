JS Origins (1)

  • 1992, first name: Oak (Sun & FirstPerson)

  • 1995: Mocha, later named LiveScript

  • Initially it was intended to be used on the server side too

  • Netscape changed LiveScript’s name to JavaScript (Java was becoming very popular back then)

JS Origins (2)

  • Microsoft decided to put more resources into a competing browser named Internet Explorer

  • After Netscape Navigator 3 was released, Microsoft introduced IE 3 with a JavaScript implementation called JScript

  • Microsoft’s implementation of JavaScript meant that there were two different JavaScript versions floating around

JS Origins (3)

  • The industry decided that the language had to be standardized.

  • 1997, JavaScript 1.1 was submitted to the European Computer Manufacturers Association (ECMA) as a proposal.

  • TC39 was assigned to “standardize the syntax and semantics of a general purpose, cross-platform, vendor-neutral scripting language"

  • 1998 the ISO/IEC adopted ECMAScript as a standard (ISO/IEC-16262). Since that time, (some) browsers have tried to use ECMAScript as a basis for their JavaScript implementations.

The BOM

  • Primarily, the BOM deals with the browser window and frames, but generally any browser-specific extension to JavaScript is considered to be a part of the BOM. The following are some such extensions

BOM extensions

  • The capability to pop up new browser windows

  • The capability to move, resize, and close browser windows

  • The navigator object, which provides detailed information about the browser

  • The location object, which gives detailed information about the page loaded in the browser.

  • The screen object, which gives detailed information about the user’s screen resolution

  • Support for cookies

  • Custom objects such as XMLHttpRequest and Internet Explorer’s ActiveXObject
