<Script> (click to go to w3schools)

  • The primary method of inserting JavaScript into an HTML page is via the <script> element

  • There are six attributes for the <script> element
    • async
    • defer
    • charset
    • src
    • type
    • language which is deprecated

Script attr (1)

  • async

    Optional. Only for external script files. It indicates that the script should begin downloading immediately but should not prevent other actions on the page such as downloading resources or waiting for other scripts to load/execute (nor itself).

  • defer

    Optional. It indicates that the execution of the script can safely be deferred (postponed) until after the document’s content has been completely parsed and displayed. Valid only for external scripts.

  • charset

    Optional. The character set of the code. (example: charset="UTF-8")

Script attr(2)

  • src

    Optional. Indicates an external file that contains code to be executed. (example: src="myscripts.js") link to w3schools.

  • type

    Optional. Indicates the content type (also called MIME type) of the scripting language being used by the code block. Traditionally, this value has always been “text/javascript”, though both “text/javascript” and “text/ecmascript” are deprecated. The type attribute is still typically set to “text/javascript” by convention and for maximum browser compatibility. This attribute is safe to omit, as “text/javascript” is assumed when missing. link to w3schools

Ways to use the <script> element (1)

  • <script type=”text/javascript”>
    function sayHello(){
        alert("Hello!");
    }
</script>

  • The JavaScript code contained inside a <script> element is interpreted from top to bottom.

  • Don't insert into javascript html code things like </script> because your script could stop working.
    Escape the character using: <\/script>

The other way to use it is linking an external file: 

  • <script type="text/javascript" src="example.js"></script>

  • Do not forget to write the </script> instead of   ..." src="example.js"/>, which is invalid html

  • Don't insert into javascript html code things like </script> because your script could stop working.

Inline code vs external files

  • It's considered best practice to include as much JavaScript code using external files

  • Why?
    • Maintainability
    • Caching
    • Future-proof

Maintainability (External files)

  • It is much easier to have a directory for all JavaScript files so that developers can edit JavaScript code independent of the markup in which it’s used.

Caching (External files)

  • Browsers cache all externally linked JavaScript files according to specific settings, meaning that if two pages are using the same file, the file is downloaded only once.

Future-proof (External files)

  • By including JavaScript using external files, there’s no need to use the XHTML or comment hacks.

XHTML Changes

  • Extensible HyperText Markup Language, or XHTML, is a reformulation of HTML as an application of XML.

  • In order to have JavaScript code into valid XHTML version you have to wrap it into a CDATA section.

  • CDATA sections are used to indicate areas of the document that contain free-form text not intended to be parsed. 

  • <script>
//<![CDATA[
    code....
//]]>
</script>

Grammar (6) - Identifiers

An Identifier (or name) is the name of a variable, a function, a property or function argument.

  • First character must be a letter, an underscore (_), or a dollar sign ($). Grammar (7) - Global and Local variables

Using the var operator to define a variable makes it local to the scope in which it was defined.

  • function test(){
    var message = "hi"; //local variable
}
test();
alert(message); //error! Try it in the console and in a file!

  • function test(){
    message = "hi"; //global variable
}
test();
alert(message); // Try it in the console and in a file

Grammar (8) - Global and Local variables (2)

JavaScript does not have block statement scope

  • if(true){
    var x = 5;
}
console.log(x); // look at the console

Another unusual thing about variables in JavaScript is that you can refer to a variable declared later, without getting an exception. This concept is known as hoisting; variables in JavaScript are in a sense "hoisted" or lifted to the top of the function or statement. However, variables that aren't initialized yet will return a value of undefined.

Grammar (9) - Global and Local variables (2)

JavaScript does not have block statement scope

  • if(true){
    var x = 5;
}
console.log(x); // look at the console

Because of hoisting, all var statements in a function should be placed as near to the top of the function as possible. This best practice increases the clarity of the code.

Global variables are in fact properties of the global object. In web pages the global object is window, so you can set and access global variables using the window.variable syntax. Consequently, you can access global variables declared in one window or frame from another window or frame by specifying the window or frame name.
